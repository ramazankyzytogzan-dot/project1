# AWS EKS ортасындағы үш деңгейлі DevSecOps жобасы

Бұл жобада мен **AWS** бұлттық платформасында ReactJS, NodeJS және MongoDB негізіндегі қосымшаны орналастырудың толық автоматтандырылған (End-to-End) жүйесін құрдым. Жұмыс барысында инфрақұрылымды баптау, қауіпсіздікті тексеру және мониторинг орнату кезеңдерін толық іске асырдым.

---

## Пайдаланылған құралдар (Tech Stack)

### Инфрақұрылым
* **Terraform**: AWS ресурстарын (VPC, EC2, EKS) код арқылы автоматты түрде құру.
* **AWS EKS**: Managed Kubernetes кластерін басқару.

### CI/CD және Қауіпсіздік
* **Jenkins**: Негізгі автоматтандыру пайплайны.
* **Ansible**: Серверлерді конфигурациялау.
* **SonarQube**: Код сапасын статикалық талдау.
* **Trivy**: Контейнерлер мен файлдарды қауіпсіздікке сканерлеу.
* **Docker & AWS ECR**: Қосымшаны контейнерлеу және бейнелерді (images) сақтау.
* **ArgoCD**: Қосымшаны кластерге GitOps принципімен орналастыру.

### Мониторинг
* **Prometheus & Grafana**: Кластер мен қосымшаның күйін бақылау.

---

## Орындалған жұмыс кезеңдері

1.  **Инфрақұрылымды құру**: Terraform көмегімен AWS-те VPC, EC2 және EKS кластерін дайындадым.
2.  **Автоматтандыру**: Jenkins арқылы кодты тексеру, жинау және сканерлеу қадамдарын қамтитын пайплайн жаздым.
3.  **Қауіпсіздік**: Trivy мен SonarQube құралдарын қолданып, код пен контейнерлердегі қателерді тексердім.
4.  **Деплой**: ArgoCD арқылы Kubernetes-ке автоматты түрде орналастыру процесін реттедім.
5.  **Мониторинг**: Жүйенің жұмысын қадағалау үшін Prometheus пен Grafana-ны баптадым.

## Жобаның каталогтары

| Қапшық | Сипаттамасы |
| :--- | :--- |
| **Application-Code** | Қосымшаның бастапқы кодтары (Frontend/Backend). |
| **Jenkins-Pipeline-Code** | Jenkins Pipeline сипатталған кодтар. |
| **Jenkins-Server-TF** | Инфрақұрылымды құратын Terraform файлдары. |
| **Kubernetes-Manifests-file** | YAML манифестері (Deployment, Service, Ingress). |
| **assets** | Схемалар мен скриншоттар. |

---

## Іске қосу нұсқаулығы
1. Репозиторийді клон жасаңыз.
2. `Jenkins-Server-TF` ішіндегі файлдар арқылы AWS ортасын құрыңыз.
3. Jenkins пайплайнын іске қосып, деплой процесін қадағалаңыз.
