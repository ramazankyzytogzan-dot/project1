# AWS EKS ортасындағы үш деңгейлі DevSecOps жобасы

Бұл жоба **AWS EKS (Elastic Kubernetes Service)** инфрақұрылымында ReactJS, NodeJS және MongoDB негізіндегі үш деңгейлі веб-қосымшаны автоматтандырылған түрде орналастыру процесін қамтиды. Жобаның басты ерекшелігі — **Gemini AI** нейрожелісі мен **n8n** платформасы арқылы интеллектуалды мониторинг жүйесін енгізу.

---

## Технологиялық стек

###  Инфрақұрылым (IaC)
* **Terraform**: Инфрақұрылымды код арқылы басқару.
* **AWS CLI**: AWS сервистерімен жұмыс істеу.
* **AWS EKS**: Kubernetes кластерін басқару.

###  CI/CD және автоматтандыру
* **Jenkins**: Pipeline-as-code концепциясы.
* **SonarQube**: Статикалық код талдауы.
* **Docker**: Контейнерлеу.
* **AWS ECR**: Контейнерлер бейнесін сақтау.
* **ArgoCD**: GitOps принциптері бойынша Continuous Delivery.

###  Мониторинг және Жасанды Интеллект
* **Prometheus & Grafana**: Мәліметтерді жинау және визуализация.
* **n8n**: Автоматтандыру платформасы.
* **Google Gemini AI**: Логтарды талдау және шешімдер ұсыну.

---

## Интеллектуалды мониторинг жүйесі (AI Integration)

Жоба аясында мониторинг деректерін талдаудың келесі алгоритмі іске асырылды:

1.  **Деректерді жинау**: Prometheus жүйесі кластердегі ақауларды немесе шекті мәндерден ауытқуды анықтайды.
2.  **Интеграция**: n8n платформасы HTTP Webhook арқылы деректерді қабылдап, оны Google Gemini AI моделіне жолдайды.
3.  **AI Талдау**: Нейрожелі техникалық логтарды талдап, ақаудың себебі мен оны шешу жолдарын (қазақ, орыс немесе ағылшын тілдерінде) тұжырымдайды.
4.  **Хабарлама**: Қорытынды есеп Telegram мессенджеріне жедел хабарлама түрінде жіберіледі.

---

## Каталогтар құрылымы

| Қапшық атауы | Сипаттамасы |
| :--- | :--- |
| **Application-Code** | Веб-қосымшаның бастапқы коды (Frontend және Backend). |
| **Jenkins-Pipeline-Code** | Кодты автоматты түрде жинауға, тестілеуге және Docker бейнелерін жасауға арналған скрипттер. |
| **Jenkins-Server-TF** | Jenkins серверін AWS-те орналастыруға арналған Terraform кодтары. |
| **Kubernetes-Manifests-file** | Kubernetes объектілерін (Deployment, Service, Ingress) сипаттайтын YAML файлдары. |

---

## Жұмысты бастау
1. Репозиторийді клон жасаңыз.
2. `Jenkins-Server-TF` арқылы серверді дайындаңыз.
3. Jenkins арқылы пайплайндарды іске қосыңыз.
4. ArgoCD арқылы қосымшаны кластерге бағыттаңыз.
